# reference_review_diy — 论据-引文-评判审核技能

对学术专著、论文、报告等长文进行系统性的**论据-引文-评判**审核。提取正文中的关键观点/数据主张，验证其引用文献的真实性，评估引文对该主张的支持程度，最终输出结构化的审核报告。

---

## 触发条件

用户说以下任意一句即加载此技能：

- "审核参考文献" / "审核引用" / "审核引文"
- "论据-引文-评判" / "reference review"
- "验证参考文献是否支持观点"
- "检查引文真实性"
- "review references" / "audit citations"

## 核心工作流

### Phase 1：范围确定

确认审核范围：

- **全文审核**：所有章节，全量引用
- **抽样审核**：每个文件抽取N条核心论据（默认15-20条/章节）
- **重点审核**：仅审核高风险的引用类型（低知名度作者、精确数字主张、跨领域延伸）

### Phase 2：文件解析

对每个待审核文件执行：

1. **Read** 读取完整文件
2. 从正文中提取所有带 `[数字]` 引用标记的**关键论点或数据主张**
3. 找到文件末尾"参考文献"部分对应的完整条目
4. 优先选择以下类型的主张：
   - 数据型主张（如"存活率仅1-5%""FMA改善+8.7分"）
   - 定义型主张（如"NSC具备自我更新和多向分化潜能"）
   - 因果型主张（如"BDNF引导轴突芽定向生长"）
   - 引自低知名度作者或非顶级期刊的主张

### Phase 3：引文验证

对每条论据-引文配对：

1. 用 **WebSearch** 搜索："第一作者 + 期刊名 + 年份 + 标题关键词"
2. 判定三类结果：
   - ✅ **真实**：搜索结果返回同一作者/同一年份/同一期刊/相似标题的论文
   - ❌ **虚假**：完全找不到该论文
   - ⚠️ **有误**：论文存在但期刊/年份/卷期/页码有误

### Phase 4：支持度评判

对每条**真实**的引文，评估其对正文主张的支持程度：

| 评级 | 含义 | 判定标准 |
|:---:|------|------|
| ★★★★★ | 完全支持 | 引文直接包含正文主张的精确数据或结论 |
| ★★★★ | 主要支持 | 引文整体方向一致，但具体数字有微小差异 |
| ★★★ | 部分支持 | 引文部分支持，但不是主张的主要来源 |
| ★★ | 弱支持 | 引文主题相关，但不直接支持该具体主张 |
| ★ | 不支持 | 引文与主张明显无关 |

### Phase 5：报告生成

输出审核报告：

```markdown
# [文档名] 论据-引文-评判审核报告

## 审核概要
- 审核范围：XX章，XX条论据
- 引文真实性：XX%真实
- 引文-论据匹配度：XX%为★★★★★或★★★★

## 各章节详细评判

### 第X章 [章节名]

#### 论据1：[简短描述观点/数据]
- **正文位置**：§X.X [引用编号]
- **引文**：[完整参考文献，作者/标题/期刊/年份/卷期/页码]
- **引文真实性**：✅ / ❌ / ⚠️
- **引文支持度**：★★★★★~★
- **评判备注**：[说明验证过程和评判理由]

## 发现问题汇总
| # | 章节 | 问题类型 | 严重程度 | 建议 |
|---|:---:|------|:---:|------|
| 1 | ChX | 引文-主张不匹配 | 紧急 | 替换为XXX |

## 综合建议
[高/中/低优先级修正建议]
```

## 多Agent并行策略

对于大型文档（>10万字），使用多Agent并行审核：

```
Agent 1 → 上篇（第1-3章）
Agent 2 → 中篇（第4-7章）
Agent 3 → 下篇（第8-10章+附录）
```

每个Agent独立执行Phase 2-4，最后人工（或主Agent）合并Phase 5。

Agent提示词模板：

```
你的任务：对专著[篇章名]进行**论据-引文-评判**的系统梳理。

文件：[文件列表]

工作流程：
1. Read读取每个文件
2. 提取所有关键观点/数据的引用标记
3. 找到对应参考文献完整条目
4. WebSearch验证引文真实性
5. 评判引文支持度（★★★★★~★）

输出：[报告文件路径.md]

每个文件抽15-20条最重要论据。直接执行，不要询问。
```

## 常见问题模式

### 高风险引用特征（优先审核）

| 特征 | 风险 | 示例 |
|------|:---:|------|
| 精确百分比（如"15%→40%"） | 数字可能来自二次引用 | "共注射Noggin将神经元分化从15%提升至40%" |
| 跨物种数据比较 | 可能无直接比较文献 | "猴脑中轴突延伸仅1mm vs 小鼠3-5mm" |
| 精确倍数（如"5.5倍""100-1000倍"） | 综述引用无法定位原始数据 | "轴突密度增加5.5倍" |
| 法规/政策引用 | 名称或施行日期易出错 | "国务院第818号令，2026年5月1日施行" |
| 经济/成本数据 | 可能来自灰色文献或估算 | "自体iPSC-NSC每位患者$150K-250K" |
| 跨领域延伸（如AI→NSC） | 引文可能不包含NSC相关内容 | "深度学习用于NSC疗效预测" |

### 常见错配模式

| 模式 | 描述 | 修正方向 |
|------|------|------|
| **体外→体内跨越** | 体外机制研究被用于支撑体内定量数据 | 补充体内验证文献 |
| **综述→原始数据错配** | 综述引用被用于支撑精确原始数据 | 标注为"综合参考文献"或追踪原始来源 |
| **单物种→多物种外推** | 单物种数据被用于多物种比较 | 拆分引用或标注外推 |
| **方法学→应用错配** | 方法学论文被用于支撑具体应用数据 | 补充应用研究文献 |
| **跨领域嫁接** | 非NSC领域论文被用于支撑NSC主张 | 标注"概念延伸"或替换 |

## 使用示例

用户输入：
> @reference_review_diy 审核这本专著的参考文献，重点看数据型主张的引文是否可靠

执行：
```
Phase 1：全文审核，14章，抽样策略
Phase 2：启动3个Agent并行解析
Phase 3：每条引文WebSearch验证
Phase 4：★★★★★~★评判
Phase 5：输出分报告→合并总报告
```

---

## 输出规范

- 报告使用Markdown格式，中文
- 每个论据独立编号
- 引文条目使用标准学术格式
- 发现的问题按严重程度（紧急/高/中/低）分级
- 评判备注简明扼要，≤50字
- 综合建议按优先级排列
